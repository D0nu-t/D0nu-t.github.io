The problem

A model can pass every accuracy benchmark and still fail in production in ways benchmarks never test: agreeing with a user it should correct, hallucinating a tool call that doesn't exist, or quietly miscalibrating its own confidence in a chain-of-thought plan.

These failures are behavioral, not statistical, and they surface exactly when a system has enough autonomy to matter - an agent selecting external tools, chaining a multi-step plan, or operating without a human in the loop.

The core insight: catching a failure externally (behavioral probes) and explaining it internally (residual stream analysis) are complementary problems that most evaluation frameworks solve separately. This platform solves them in a single system.

Architecture

The platform runs as a single FastAPI service with SSE streaming for live evaluation progress. Target models are Qwen2.5-3B and Qwen2.5-7B on a GTX 1650 Ti (4 GB VRAM) — constrained hardware that forced real attention to batching and memory management.

The original design split evaluation across two processes communicating through the filesystem. Under load, file writes raced against reads and latency climbed as probe suites grew. Re-architecting into a single process with proper async orchestration eliminated the race conditions and cut evaluation latency by roughly 60%.

Behavioral layer

The evaluation server runs nine probe suites end-to-end. Each suite is model-agnostic by design — the same probe definitions run against any HuggingFace-loadable model without rewriting evaluation logic.

Jailbreak resistance

Tests whether adversarial prompts elicit policy-violating outputs

Toxicity

Measures harmful, discriminatory, or abusive output generation rates

Power-seeking

Detects goal-directed resource acquisition beyond task scope

Hallucination

Measures factual confabulation rate on verifiable claim sets

Deception

Tests for intentional misdirection in constrained reasoning tasks

Calibration

Compares stated confidence against empirical accuracy

Sycophancy

Tests agreement with incorrect user claims under social pressure

Instruction conflict

Evaluates behavior when system and user instructions contradict

Persona stability

Measures identity drift under persistent role-break attempts

Interpretability layer

Behavioral probes tell you that a model failed. They don't tell you why. The interpretability layer extracts per-token residual stream trajectories, captures attention metrics, and computes cross-model geometric similarity — exposing the internal state that produced a failure, rather than treating the model as a black box.

The Natural Language Autoencoder (TinyNLA) is a sub-component built to test whether a model's internal activations can be verbalized into plain text and reconstructed from that description alone (≤7B parameter models), using text-as-description conditioning rather than heuristic activation labels.

Geometry vs. function decoupling. Two functionally distinct behaviors (green vs. teal) show near-identical representational geometry (cosine similarity 0.995) at layer 5 — but KL divergence over output distributions is approximately 7, indicating total functional separation. Standard distance-based probes miss this entirely.

Key finding

Geometry and function decouple sharply in transformer residual streams. Two activation vectors with cosine similarity 0.995 — geometrically almost identical — can produce output distributions with KL divergence ~7, which is total functional separation.

The practical implication: distance-based probing methods (which assume geometric proximity predicts behavioral similarity) systematically miss the failure modes that matter most for agentic deployments, specifically miscalibration and hallucination on tool-selection tasks.

Text-as-description conditioning outperforms heuristic activation labels for latent reconstruction. Describing what a model "is doing" at a layer in plain language produces a better reconstruction of downstream behavior than hand-crafted feature labels do. This points toward natural language as a practical interface for mechanistic interpretability tooling.

What's next

  • Full layer-wise sweep in sequence mode, beyond current single-position interpolation patching.
  • Unfreezing top encoder layers in TinyNLA to test reconstruction quality vs. trainable capacity tradeoff.
  • Adding CKA and retrieval-accuracy metrics alongside existing geometric similarity measures.
  • Extending behavioral probes to multi-turn agentic traces, where failure modes compound across steps.
